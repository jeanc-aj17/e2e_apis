# Conclusiones del ejercicio de automatización E2E

- El flujo de compra en SauceDemo fue automatizado exitosamente utilizando Cypress.io, validando desde el login hasta la confirmación final del pedido.
- Cypress demostró ser una herramienta eficiente para pruebas E2E, con una sintaxis clara, ejecución rápida y excelente visibilidad de los pasos.
- La estructura del sitio permitió una fácil identificación de elementos mediante atributos `data-test`, lo que simplificó la escritura de selectores robustos.
- Se aplicaron buenas prácticas como validación de mensajes visibles y simulación de interacción real del usuario.
- El test es fácilmente reproducible en cualquier entorno con Node.js, sin necesidad de configuraciones complejas.
- El flujo cubre aspectos clave de automatización funcional: autenticación, navegación, interacción con formularios, validación de resultados.
- Se utilizó un enfoque directo y confiable, evitando sobreingeniería y priorizando la claridad del código.
- El ejercicio permitió reforzar habilidades en Cypress, DOM traversal, y validación de contenido dinámico.
- Se identificó que el mensaje final esperado no era “CHECKOUT: COMPLETE!” sino “THANK YOU FOR YOUR ORDER”, lo que requirió ajustar la validación para reflejar el comportamiento real del sitio.
- Como mejora futura, se podría implementar:
  - Separación por Page Objects para mayor escalabilidad.
  - Uso de datos parametrizados para pruebas con múltiples usuarios.
  - Integración con herramientas de reporte como Mochawesome o Allure.

  - Ejecución en pipelines CI/CD para validación continua.

La carpeta 'node_modules' tuve que quitarla para poder subir eje ejercicios al repositorio de Gihub
